Tags

Tags are major part of HTML. A tag directs the browser on how it should show the different content of a website.
Like Text formatting or showing images.

Tags have the general format of

<TagLettersOrComand>Content/Text< /TagLettersOrComand>


There are two different syntax from the standard tags;
  • Void
    • Void tags don't require a closing, example the break line tag <br> doesn't require: 
    </br>
  • Attributes
    • Attribute defines extra options for the tag or provide needed information. Some attributes are Mandatory (required) & some are Optional.
    <Tag attribute="value"> Content </Tag>
There are two types of tags
  1. Inline
  2. Block

Inline Tag Keeps the text on same line. It doesn't add extra white spaces. It's best understood when compared with the Block type.

Block Tag It has the effect of surrounding the tagged text with invisible box that has width and height. It separate the text from surrounding text by providing white space through the borders of the invisible box.

Two famous tags, DIV and SPAN, are used as containers for other HTML elements. They are used mostly with CSS.
DIV is a Block tag. SPAN is an inline tag.

World Wide Web

The world Wide Web (www) is a collection of HyperText Markup Language pages (HTML) on millions of computers.

To connect to the net you need a browser (like Chrome or Firefox) which interpret and show HTML pages properly and formatted. The browser communicate with servers through out the world through Hypertext Transfer Protocol (HTTP); which is a simple protocol for HTML transfer. The servers deliver back to your browser the requested page and information.



Programmers 5 ways of thinking:

Procedural Thinking:

What are the procedures that need to be followed to achieve a certain task. Like for hand to pick up a pencil and write with it, though it looks simple it requires a lot of procedures: finding the pencil, coordinate the eye with hands to get the pencil, holding the pencil with proper force, identifying its shape (tip, end, length), rotating the pencil in a certain way, finding papers, finding starting point of writing, applying proper pressure on thetip … etc

Abstract Thinking:

To see similarities in things allowing some tools to be reused rather than re-inventing the wheel. Abstract thinking is clear in websites designs through CSS.

System thinking:

Is linking the ideas (like the procedures) in an organised way to know how to go about it, where to start and what to do next… I feel it is like a project plan.

Machine Empathy:

To understand how the computer work. How the computer interpret the commands. What is its capabilities language wise.

Debugging:

To have a systematic way to find mistakes and toggle bugs. Usually it contains these steps:
  • What is the symptoms of the mistake; the actions, messages, behavior it is creating.
  • In what stage it happen (approximate location if possible).
  • Putting theories about how this happen.
  • Testing the theories.
  • Fixing the mistake
